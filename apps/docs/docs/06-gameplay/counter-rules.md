# Правила контра и граничные кейсы

Этот документ фиксирует правила контра и набор тест‑кейсов для проверки поведения.

## Основные правила

- Контр — направленная связь, работает только по направлению ребра.
- Контр‑умения активируются только если `get_relation(source, target) == counter`.
- Союз с самим собой (`ally`) не даёт контр‑доступа.
- Торговая связь (`trade`) не является контром и не включает контр‑умения.

## Эталонная матрица (проверка)

| SOURCE → TARGET | FOXY | OXY | BEAR | OWL |
| --- | --- | --- | --- | --- |
| FOXY | ally | counter | neutral | neutral |
| OXY | neutral | ally | counter | neutral |
| BEAR | counter | neutral | ally | neutral |
| OWL | trade | trade | trade | ally |

## Граничные кейсы

1. Source и target совпадают — всегда `ally`.
2. OWL против OWL — `ally`, контр‑умения недоступны.
3. OWL против любого — `trade`, контр‑умения недоступны.
4. Контр работает только в одну сторону, обратная связь всегда `neutral` или `trade`.
5. Входящие запросы без архетипа отклоняются как `neutral` с ошибкой валидации.

## Набор тестов (минимум)

1. Применение контр‑умения при `counter` проходит и создаёт событие.
2. Применение контр‑умения при `neutral` или `trade` блокируется.
3. Само‑таргетинг контр‑умений запрещён.
4. Таймер эффекта корректно истекает и публикует `effect_expired`.
5. Повторный `event_id` не создаёт дубликаты.
