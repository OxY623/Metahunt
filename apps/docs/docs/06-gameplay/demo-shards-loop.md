# Демо-схема Осколков

:::note Статус
Предложение для быстрой демо-версии. Цель — за 10-15 минут показать игроку выбор, конфликт, выгоду и риск.
:::

## Главная идея

Осколки должны начисляться не за пассивный скролл, а за действия, которые двигают социальную игру: вход, сообщение, квест, сделку, контр-ход, приглашение. Игрок быстро понимает:

- активность дает ресурс;
- сильные действия стоят ресурса;
- фракция меняет способ заработка;
- чужие действия создают возможности для тебя.

В демо не нужна сложная экономика. Нужна короткая петля:

```
Войти -> выбрать фракцию -> выполнить действие -> получить Осколки ->
потратить на фракционный ход -> увидеть ответ другой фракции
```

## Демо-баланс на первые сутки

| Параметр | Значение |
| --- | --- |
| Стартовый баланс | 80 Осколков |
| Дневной кап демо | +100 Осколков |
| Мягкий кап после лимита | награды x0.5 |
| Минимальная цена сильного действия | 10 Осколков |
| Максимальная цена сильного действия | 40 Осколков |
| Целевой баланс игрока | 20-160 Осколков |

Почему 80 на старте: игрок сразу может попробовать 2-4 действия, но не может бесконечно давить других.

## Начисление Осколков

| Событие | Награда | Ограничение | Зачем это нужно |
| --- | ---: | --- | --- |
| Первый вход за день | +10 | 1 раз в день | Возврат без гринда |
| Первое сообщение в комнате | +5 | 1 раз в день | Мягко ведет в чат |
| Завершить быстрый квест | +25 | 3 раза в день | Основной заработок демо |
| Успешный контр-ход | +12 | 5 раз в день | Поощряет фракционную игру |
| Ответить на действие против себя | +8 | 5 раз в день | Не дает жертве выпадать |
| Использованный инвайт | +20 до налога | 3 раза в день | Дает вирусный рост |
| Сделка с OWL | +15-35 | 3 раза в день | Создает рынок информации |
| Пост с 3 реакциями | +10 | 2 раза в день | Связывает постинг и игру |
| Гео-чек в активной зоне | +8 | 3 раза в день | Подключает карту |

Анти-абуз правило для демо: повтор одного и того же типа награды после лимита не блокируется полностью, но дает только 50%. Так игрок не упирается в стену, но понимает, что выгоднее разнообразить действия.

## Траты в демо

| Действие | Цена | Эффект |
| --- | ---: | --- |
| OXY: Маркер охоты | 10 | Помечает цель на 3 минуты |
| OXY: Прямой удар по BEAR | 15 | Снимает часть темпа/XP цели |
| FOXY: Вуаль | 12 | Скрывает присутствие на 2 минуты |
| FOXY: Глитч по OXY | 18 | Мешает цели 30 секунд |
| BEAR: Золотой щит | 25 | Иммунитет к контр-скиллам на 5 минут |
| BEAR: Блокировка порта по FOXY | 30 | Блокирует активные умения цели на 90 секунд |
| OWL: Шепот | 15 | Анонимный DM в комнате |
| OWL: Интел-скан | 20 | Показывает активность без точных координат |
| Усилить пост | 20 | Поднимает пост в видимости комнаты |
| Купить энергию | 30 | +60 энергии |

Для демо лучше держать цены ниже текущего строгого режима, чтобы игрок увидел эффект, а не только витрину.

## Как работают фракции между собой

Базовая матрица остается прежней:

| Кто действует -> цель | FOXY | OXY | BEAR | OWL |
| --- | --- | --- | --- | --- |
| FOXY | ally | counter | neutral | neutral |
| OXY | neutral | ally | counter | neutral |
| BEAR | counter | neutral | ally | neutral |
| OWL | trade | trade | trade | ally |

Но для интереса игроку нужно показать не матрицу, а простые мотивы.

### OXY

OXY зарабатывает на охоте и ответных действиях. Ему интересно искать BEAR, потому что контр-ходы дают бонусные Осколки и статус давления.

Демо-петля:

```
увидел BEAR в комнате -> поставил Маркер охоты -> сделал Прямой удар ->
получил +12 за контр-ход -> стал целью для FOXY
```

### FOXY

FOXY зарабатывает на скорости, инвайтах и налоговой доле. Ей интересно провоцировать OXY и уходить из-под BEAR.

Демо-петля:

```
заметила OXY -> включила Вуаль -> дала Глитч ->
получила +12 за контр-ход -> рискует получить блокировку от BEAR
```

### BEAR

BEAR зарабатывает на налогах и контроле. Ему интересно держать экономику, а не просто банить всех.

Демо-петля:

```
видит FOXY -> ставит Блокировку порта -> получает +12 за контр-ход ->
параллельно собирает налог с инвайтов и визитов
```

### OWL

OWL не контрит напрямую. Она зарабатывает на информации: кто кого атаковал, кто копит ресурс, кто под щитом, где активная зона.

Демо-петля:

```
собрала сигнал -> продала подсказку OXY или FOXY -> получила Осколки ->
создала новую интригу между фракциями
```

## Фракционный круг интереса

```
FOXY давит OXY
OXY давит BEAR
BEAR давит FOXY
OWL продает всем информацию
```

Это дает понятную драму:

- OXY хочет справедливости и охоты.
- FOXY хочет выгоды и хаоса.
- BEAR хочет контроля и налога.
- OWL хочет знать больше всех и продавать момент.

## Массовое взаимодействие игроков

Когда пользователей становится много, игра не должна превращаться в случайные дуэли. Для демо используем модель фракционного пульса:

| Слой | Что считает система | Что видит игрок |
| --- | --- | --- |
| Сила фракции | Сколько игроков выбрали FOXY/OXY/BEAR/OWL | Процент сети и численность |
| Линия давления | FOXY -> OXY -> BEAR -> FOXY | Кого выгодно атаковать сейчас |
| Рынок данных | OWL -> все фракции | Где можно купить/продать сигнал |
| Активные пары | min(игроки source, игроки target) | Сколько потенциальных конфликтов доступно |
| Рекомендация | Архетип текущего пользователя | Следующее полезное действие |

Так 4 архетипа масштабируются на 40, 400 и 4000 игроков: человек играет не против абстрактной толпы, а внутри понятной политической карты.

### Правило комнат

- В общей комнате видны все фракции и общий пульс.
- Во фракционной комнате игроки координируют свою сторону.
- Навык выбирает цель кликом по сообщению, но решение подсказывает пульс.
- Если одна фракция слишком сильная, OWL получает больше поводов продавать данные против неё.

### Почему это интересно клиентам продукта

- Появляется понятная командная динамика, а не просто чат с валютой.
- Баланс виден в интерфейсе: кто усиливается, кто под давлением, где рынок информации.
- Любой новый пользователь сразу понимает свою роль: давить, защищать, провоцировать или продавать.
- Система создаёт повод вернуться: фракционный расклад меняется, значит меняются цели.

## Налоги для демо

Налоги должны быть видимыми, иначе игроки их не почувствуют.

| Событие | Налог | Куда идет |
| --- | ---: | --- |
| Использован инвайт | 20% | BEAR treasury |
| OXY входит в комнату FOXY | 10 Осколков | 70% BEAR, 30% FOXY |
| Покупка разбана | 40 Осколков | 100% BEAR |
| Сделка через OWL | 10% | OWL fee |

В UI после действия нужно показывать короткую строку:

```
+20 инвайт, -4 налог BEAR, итог +16
```

Так экономика выглядит живой, а не скрытой бухгалтерией.

| Квест | Для кого | Условие | Награда |
| --- | --- | --- | ---: |
| След в комнате | Все | Написать первое сообщение | +5 |
| Первый ход | Все | Использовать фракционный скилл | +15 |
| Ответная искра | Все | Ответить на действие против себя | +8 |
| Охота на власть | OXY | Применить контр к BEAR | +12 |
| Красивый обман | FOXY | Применить контр к OXY | +12 |
| Печать контроля | BEAR | Применить контр к FOXY | +12 |
| Тихая цена | OWL | Завершить сделку | +20 |
| Голос района | Все | Получить 3 реакции на пост | +10 |

## Что показать клиенту на демо

1. Игрок выбирает фракцию и получает 80 Осколков.
2. В комнате видит сообщения других фракций.
3. Кликает по сообщению и видит доступные действия.
4. Делает действие, получает событие в чат и запись в ledger.
5. Видит, как часть Осколков ушла налогом или комиссией.
6. Получает ответное действие от другой фракции.
7. Понимает: фракция не скин, а правила поведения.

## Предложения к реализации

### 1. Сделать ledger главным источником правды

Любое начисление и списание идет через `shards_ledger`. Даже демо-начисления должны иметь `reason`, `delta`, `balance_after`, `meta`.

Минимальные причины:

| Reason | Когда используется |
| --- | --- |
| `daily_login` | Ежедневный вход |
| `quest_reward` | Быстрый квест |
| `counter_reward` | Успешный контр-ход |
| `reaction_reward` | Пост набрал реакции |
| `invite_reward` | Использован инвайт |
| `skill_cost` | Списание за скилл |
| `tax` | Налог BEAR или комиссия OWL |
| `refund` | Откат при ошибке действия |

### 2. Ввести фракционные модификаторы

Для демо достаточно простых бонусов:

| Фракция | Бонус |
| --- | --- |
| OXY | +20% к награде за успешный контр против BEAR |
| FOXY | +1 дневной инвайт и 30% доля с налогов BEAR |
| BEAR | получает налоги и скидку 20% на щит |
| OWL | получает 10% комиссии со сделок и видит больше причин ledger в UI |

### 3. Добавить "событие-провокацию" каждые 5 минут

Если в комнате мало людей, система создает мягкий повод:

| Событие | Эффект |
| --- | --- |
| Слух от OWL | Подсвечивает игрока с большим балансом |
| Проверка BEAR | На 2 минуты повышает налог в комнате |
| След FOXY | Скрывает автора одного поста |
| Метка OXY | Показывает активную цель для охоты |

Так демо не умирает, даже если тестируют 2-3 человека.

### 4. Сделать все эффекты короткими

Для демо таймеры должны быть быстрыми:

| Тип эффекта | Длительность |
| --- | ---: |
| Глитч | 30 секунд |
| Блокировка | 60-90 секунд |
| Вуаль | 2 минуты |
| Маркер | 3 минуты |
| Щит | 5 минут |

Короткие эффекты лучше показывают продукт и не превращают демо в наказание.

### 5. Показывать выгоду и риск в кнопке действия

Кнопка скилла должна сразу объяснять цену:

```
Глитч -18, шанс награды +12
Шепот -15, комиссия OWL 10%
Щит -25, защита 5 мин
```

Игроку не нужно открывать правила, чтобы понять ставку.

## MVP-формула начисления

Для реализации можно использовать одну функцию:

```ts
reward = baseReward * modeMultiplier * factionModifier * antiAbuseModifier
```

Где:

| Переменная | Демо-значение |
| --- | --- |
| `baseReward` | значение из таблицы события |
| `modeMultiplier` | 1.0 для демо |
| `factionModifier` | 1.0-1.2 |
| `antiAbuseModifier` | 1.0 до капа, 0.5 после капа |

Финальное значение округляется вниз. Если результат меньше 1, начисление не создается.

## Минимальный набор для сборки демо

- Баланс Осколков в профиле.
- Ledger последних 20 операций.
- Начисления: daily login, quest, counter, invite, OWL deal.
- Списания: skill cost, tax, OWL fee.
- Матрица отношений фракций.
- Кнопки действий по клику на сообщение.
- Системные события в чат после начислений и списаний.

Этого достаточно, чтобы клиент увидел продуктовую магию: люди не просто переписываются, они играют социальными последствиями.
