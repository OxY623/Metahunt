# MVP Roadmap и Demo Script

:::note Цель
Этот документ связывает геймплей, карту, Осколки, фракции и демо-показ в один рабочий план.
:::

MetaHunt должен показываться не как набор экранов, а как короткая игровая история:

```
выбор архетипа -> осколки -> чат -> фракционный ход -> карта ->
гео-действие -> ledger -> вклад в фракцию
```

---

| Блок | Статус | Что работает |
| --- | --- | --- |
| Архетипы | Есть | FOXY, OXY, BEAR, OWL |

| Скиллы | Есть база | glitch, direct strike, shield, ban, whisper, owl deal |

| Инвайты | Есть база | лимиты, redeem, награда |
| Постинг | Есть база | посты, boost, anonymous |
| Карта | Есть база | tiles, check-in, ping, search, clusters |
| Фракционный pulse | Есть база | численность фракций, линии давления |

---

## Главная MVP-петля

Игрок должен пройти её за 10-15 минут:

1. Зарегистрироваться или войти.
2. Выбрать архетип.
3. Получить стартовые Осколки.
4. Забрать daily reward.
5. Написать первое сообщение в чат.
6. Выбрать цель кликом по сообщению.
7. Применить фракционный скилл.
8. Увидеть списание и/или награду в ledger.
9. Открыть карту.
10. Найти район через поиск.
11. Сделать check-in или ping.
12. Увидеть tile/cluster и влияние фракции.

Если этот путь работает гладко, демо уже объясняет продукт.

## Demo Script для клиента

### Сцена 1: Dashboard

Показать:

- профиль игрока;
- архетип;
- баланс Shards;
- энергию;
- последние операции ledger;
- кнопки демо-наград.

Фраза для демо:

> Здесь видно, что это не просто профиль. Это оперативная панель игрока: ресурсы, роль и последствия действий.

### Сцена 2: Выбор архетипа

Показать 4 роли:

| Архетип | Что объяснить |
| --- | --- |
| OXY | Давление и охота на BEAR |
| FOXY | Манипуляция и атака на OXY |
| BEAR | Контроль, налоги, блокировки |
| OWL | Информация, сделки, нейтралитет |

Фраза:

> Архетип не скин. Он меняет экономику, цели и доступные действия.

### Сцена 3: Чат

Показать:

- комнаты;
- сообщения разных фракций;
- выбор цели кликом;
- доступные скиллы;
- цену и награду на кнопке;
- фракционный pulse.

Фраза:

> Чат здесь не просто переписка. Каждое сообщение может стать целью, сделкой или атакой.

### Сцена 4: Осколки и ledger

Показать:

- списание за скилл;
- награду за counter;
- daily reward;
- ledger последних операций.

Фраза:

> Осколки создают выбор: потратить на давление, защиту, анонимность или информацию.

### Сцена 5: Карта

Показать:

- поиск по городу или координатам;
- кластеры;
- tile popup;
- check-in;
- ping;
- фракционный цвет/доминанту.

Фраза:

> Карта показывает, где происходит игра. Районы становятся ареной влияния фракций.

## Что нужно доделать для сильного MVP

### 1. Территориальный контроль

Сейчас есть `dominant_archetype`, но нужен полноценный score по фракциям.

Нужно:

- добавить влияние FOXY/OXY/BEAR/OWL в tile;
- начислять влияние за check-in, ping, post, skill;
- показывать шкалу влияния в popup;
- считать contested zones.

См. [Территории и Фракционная Война](./territory-roadmap).

### 2. Карта как источник Осколков

Нужно:

- награда за первый check-in дня;
- награда за contested check-in;
- награда за перехват tile;
- списания через ledger для всех map actions;
- подсказки цены прямо на кнопках.

### 3. Фракционные цели недели

Нужно:

- weekly goals для каждой фракции;
- faction score;
- leading faction;
- underdog bonus;
- вклад игрока в общий счёт.

### 4. UI “что делать дальше”

На dashboard и карте нужен блок рекомендаций:

| Состояние | Рекомендация |
| --- | --- |
| Нет архетипа | Выбрать роль |
| Нет Осколков | Забрать daily reward или сделать квест |
| Есть цель counter | Применить скилл |
| Есть contested tile | Сделать check-in/ping |
| Фракция проигрывает | Использовать underdog bonus |

### 5. Админ/демо-режим

Для показа клиентам нужен удобный режим:

- быстро создать тестовых игроков;
- назначить архетипы;
- насыпать Shards;
- создать активные tiles;
- сгенерировать сообщения в чат;
- сбросить demo state.

## Definition of Done для MVP

MVP готов к клиентскому демо, когда можно без объяснений пройти путь:

1. Игрок выбирает архетип.
2. Видит стартовые Shards.
3. Получает награду.
4. Пишет в чат.
5. Применяет скилл к подходящей фракции.
6. Видит ledger.
7. Открывает карту.
8. Ищет район.
9. Видит кластеры.
10. Делает check-in/ping.
11. Видит изменение tile.
12. Понимает вклад в свою фракцию.

Если пользователь понимает это за 10 минут, MetaHunt уже работает как продуктовая идея.
