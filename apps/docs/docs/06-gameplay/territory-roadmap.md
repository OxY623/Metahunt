# Территории и Фракционная Война

:::note Статус
Roadmap для следующего этапа после демо-экономики Осколков, геопоиска и кластеризации карты.
:::

Эта страница описывает, что ещё нужно реализовать, чтобы MetaHunt стал не просто чатом с картой, а живой социальной стратегией: игроки выбирают архетип, действуют на карте, зарабатывают и тратят Осколки, а фракции борются за районы.

---

## Главная идея

Карта должна отвечать на действия игроков.

Сейчас карта показывает активность: check-in, ping, intensity, dominant archetype, кластеры. Следующий шаг — сделать из этого территориальную игру:

```
check-in / ping / пост / скилл -> влияние в tile -> контроль района ->
фракционные бонусы -> новые конфликты -> траты Осколков
```

Игрок должен видеть:

- где его фракция сильна;
- где её давят;
- где выгодно сделать check-in или ping;
- почему его действие меняет общий расклад.

---

## Фича 1: Живая карта

### Что это

Живая карта — это карта, которая показывает не только точки активности, но и состояние борьбы:

- контроль тайлов;
- фракционные кластеры;
- всплески активности;
- опасные зоны;
- рекомендации по следующему действию.

### Что видит игрок

| UI-элемент | Что показывает |
| --- | --- |
| Цвет tile | Доминирующий архетип |
| Интенсивность | Насколько активна зона |
| Кластер | Сколько активных тайлов рядом |
| Событие зоны | Что сейчас происходит в районе |
| Рекомендация | Что выгодно сделать текущему архетипу |

### Примеры событий зоны

| Событие | Условие | Эффект |
| --- | --- | --- |
| `foxy_spike` | FOXY быстро набирает активность | OXY получает подсказку для охоты |
| `bear_checkpoint` | BEAR удерживает tile | Цена действий в tile выше |
| `oxy_hunt` | OXY давит BEAR в районе | BEAR видит угрозу контроля |
| `owl_market` | OWL часто торгует данными | В зоне появляются сделки и слухи |
| `contested_zone` | 2+ фракции близки по влиянию | Повышенные награды за действия |

### Что нужно реализовать

#### Backend

- Добавить вычисление статуса зоны:
  - `stable`
  - `contested`
  - `spike`
  - `checkpoint`
  - `market`
- Расширить `GeoTile` или добавить отдельную таблицу `geo_tile_state`.
- Добавить endpoint:

```http
GET /api/v1/map/zones?bbox=...&zoom=...
```

Ответ:

```ts
{
  items: [
    {
      tile_id: "x1439y2075",
      state: "contested",
      dominant_archetype: "FOXY",
      intensity: 0.72,
      control_score: 58,
      recommended_action: "OXY может поставить Маркер охоты"
    }
  ]
}
```

#### Frontend

- Добавить слой zone state поверх карты.
- Показывать бейдж состояния в popup tile.
- Добавить фильтр “только спорные зоны”.
- В правой панели показывать “горячие районы”.

## Фича 2: Контроль территорий

### Что это

Каждый tile получает очки влияния по фракциям. Контроль принадлежит той фракции, у которой больше влияния.

Влияние не должно быть вечным. Оно затухает, чтобы карта жила:

```
influence_after_decay = influence * decay_rate
```

### Источники влияния

| Действие | Влияние | Примечание |
| --- | ---: | --- |
| Check-in | +5 | Базовое действие |
| Ping | +8 | Стоит Осколки |
| Пост с geo_tile | +4 | Связка постинга и карты |
| Boosted post | +10 | Платное усиление |
| Успешный контр-скилл в зоне | +12 | Связка чата и карты |
| OWL deal по зоне | +6 | Информационный след |

### Фракционные модификаторы

| Архетип | Бонус на карте |
| --- | --- |
| OXY | +20% влияния за действия против BEAR |
| FOXY | +15% влияния за быстрые pings и masked следы |
| BEAR | +25% удержания уже контролируемого tile |
| OWL | Не захватывает напрямую, но усиливает contested-зоны через сделки |

### Что нужно реализовать

#### Database

Новая таблица:

```sql
geo_influence
```

Поля:

| Поле | Тип | Зачем |
| --- | --- | --- |
| `id` | UUID | ID записи |
| `tile_id` | string | Tile |
| `archetype` | string | FOXY/OXY/BEAR/OWL |
| `score` | float | Текущее влияние |
| `source` | string | checkin/ping/post/skill/deal |
| `created_at` | datetime | История |
| `decays_at` | datetime | Когда влияние начинает исчезать |

Можно начать проще: хранить агрегаты прямо в `geo_tiles`:

```ts
foxy_score
oxy_score
bear_score
owl_score
control_updated_at
```

Для демо второй вариант быстрее.

#### Backend

- Метод `add_influence(tile_id, archetype, amount, source)`.
- Пересчёт `dominant_archetype`.
- Endpoint:

```http
GET /api/v1/map/control?bbox=...
```

- Endpoint:

```http
POST /api/v1/map/control/action
```

Для специальных действий в зоне.

#### Frontend

- Покрасить тайлы по контролю фракции.
- В popup показывать шкалу:

```text
FOXY 42 | OXY 35 | BEAR 18 | OWL 5
```

- На кластере показывать доминирующий контроль района.
- В панели справа показывать:
  - “наш контроль”;
  - “под угрозой”;
  - “лёгкая цель”.

## Фича 3: Экономика Осколков на карте

### Что это

Осколки должны быть связаны с картой, чтобы гео-действия были частью экономики, а не отдельной игрушкой.

### Начисления

| Событие | Награда | Лимит |
| --- | ---: | --- |
| Первый check-in за день | +8 | 1 раз |
| Check-in в contested zone | +10 | 3 раза в день |
| Удержание tile фракцией | +15 | 1 раз в день |
| Вернуть tile из-под контра | +20 | 2 раза в день |
| OWL продала intel по зоне | +20 | 3 раза в день |

### Траты

| Действие | Цена | Эффект |
| --- | ---: | --- |
| Geo ping | 10 | Усиливает tile и показывает активность |
| Intel scan OWL | 16 | Показывает распределение влияния |
| Checkpoint BEAR | 20 | Удерживает tile и повышает стоимость атак |
| Mask trace FOXY | 11 | Скрывает фракционный след |
| Hunt mark OXY | 12 | Помечает contested target |

### Что нужно реализовать

#### Backend

- Расширить `shards_ledger.reason` для карты:

| Reason | Когда |
| --- | --- |
| `geo_checkin_reward` | Первый/полезный check-in |
| `geo_control_reward` | Удержание или перехват tile |
| `geo_ping_cost` | Платный ping |
| `geo_intel_cost` | OWL scan |
| `geo_checkpoint_cost` | BEAR checkpoint |
| `geo_mask_cost` | FOXY trace mask |
| `geo_hunt_cost` | OXY hunt mark |

- Все действия карты должны проходить через ledger.
- Добавить daily cap по карте отдельно от общего cap.

#### Frontend

- В карточке tile показывать:

```text
Ping -10, влияние +8
Intel scan -16, раскрыть баланс сил
Check-in +8 если первый за день
```

- После действия показывать строку ledger:

```text
+10 contested check-in, баланс 124
```

## Фича 4: Массовое взаимодействие 4 архетипов

### Проблема

Если архетипов всего 4, а игроков много, нельзя строить игру как дуэли “один на один”. Нужен групповой слой:

- фракции;
- районы;
- общие цели;
- вклад отдельного игрока в общий счёт.

### Решение

Каждая фракция получает коллективные цели недели.

| Фракция | Цель недели | Пример |
| --- | --- | --- |
| OXY | Сломать контроль BEAR | Забрать 10 BEAR tiles |
| FOXY | Разогнать хаос | Сделать 30 successful glitches |
| BEAR | Удержать казну | Собрать 500 tax shards |
| OWL | Продать информацию | Закрыть 40 intel deals |

### Фракционный счёт

Счёт считается не только по Осколкам:

```ts
faction_score =
  control_score * 0.4 +
  economy_score * 0.25 +
  counter_score * 0.25 +
  activity_score * 0.1
```

Где:

| Score | Что учитывает |
| --- | --- |
| `control_score` | Контроль tile/районов |
| `economy_score` | Налоги, сделки, траты |
| `counter_score` | Успешные контр-действия |
| `activity_score` | Check-ins, posts, chat |

### Что нужно реализовать

#### Database

Новая таблица:

```sql
faction_events
```

Поля:

| Поле | Тип |
| --- | --- |
| `id` | UUID |
| `archetype` | string |
| `event_type` | string |
| `score_delta` | int |
| `tile_id` | string nullable |
| `user_id` | UUID |
| `meta` | json |
| `created_at` | datetime |

#### Backend

- `record_faction_event(archetype, event_type, score_delta, meta)`.
- Endpoint:

```http
GET /api/v1/game/factions/pulse
```

Уже есть базовая версия. Нужно расширить:

- добавить weekly score;
- добавить goals;
- добавить leading faction;
- добавить “слабую сторону”.

#### Frontend

- На dashboard добавить блок “Фракционная война”.
- В чате показывать фракционный pulse.
- На карте показывать weekly objectives.

## Фича 5: Баланс слабой стороны

### Зачем

Если одна фракция начинает доминировать, остальные могут потерять интерес. Нужен мягкий comeback-механизм.

### Правила

| Условие | Бонус |
| --- | --- |
| Фракция ниже 20% контроля | +15% наград за contested actions |
| Фракция под давлением 2 циклов подряд | Скидка 10% на defensive actions |
| BEAR слишком силён | OXY получает больше hunt targets |
| FOXY слишком сильна | BEAR получает усиленные блокировки |
| OXY слишком силён | FOXY получает дешевле glitch |
| OWL мало | Сделки OWL дают +25% награды |

### Что нужно реализовать

- Функция `get_balance_modifiers(archetype)`.
- Применять модификатор в:
  - shard rewards;
  - skill costs;
  - map influence;
  - faction score.
- В UI показывать не скрытый бафф, а статус:

```text
Underdog bonus: +15% за спорные зоны
```

## Приоритет реализации

### Этап 1: Территориальный MVP

1. Добавить influence score по фракциям в `geo_tiles`.
2. Начислять влияние за check-in и ping.
3. Показывать контроль tile в popup.
4. Красить кластеры по доминирующей фракции.
5. Добавить rewards за contested check-in.

### Этап 2: Экономика карты

1. Все map actions писать в `shards_ledger`.
2. Добавить цены и награды прямо в UI кнопок.
3. Добавить OWL intel scan.
4. Добавить BEAR checkpoint.
5. Добавить FOXY mask trace и OXY hunt mark.

### Этап 3: Фракционная война

1. Добавить `faction_events`.
2. Считать weekly score.
3. Добавить goals недели.
4. Показывать “кто выигрывает” на dashboard.
5. Включить underdog modifiers.

## Минимальный Definition of Done

Фича считается готовой, когда игрок может:

1. Найти район на карте.
2. Увидеть кластеры и контроль фракций.
3. Сделать check-in или ping.
4. Получить или потратить Осколки.
5. Увидеть изменение влияния tile.
6. Понять, какую пользу он дал своей фракции.

Если эти 6 пунктов работают, карта становится не экраном навигации, а сердцем игры.
